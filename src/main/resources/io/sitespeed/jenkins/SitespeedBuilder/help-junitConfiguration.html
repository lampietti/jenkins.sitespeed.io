To fetch the JUnit XML files from sitespeed.io, add this to your build:

  • Add a Post build action
  • Choose: Publish JUnit test result report
  • Add “Test report XMLs”: junit/*.xml

Check out the documentation of how to configure the JUnit script. By default you don't have to configure anything to get it up and running but you probably want to do two things:

  • -s Skip these rule tests. A comma separated list of the key name of the rules; for example "ycsstop,yjsbottom"
  • -t Point out the XML timing limit file, the file where you set limits when a Navigation Time API timing value should fail a test.

By default the resultBaseDir (where the script fetches the data) is set to Jenkins workspace dir + /sitespeed-result/ and the output directory set to Jenkins workspace, and you can't change them running from Jenkins.
